# AI Desk Pet Runtime3D 开发任务清单 v1

## 1. 使用规则（强制）

1. 先对齐需求文档：`docs/runtime3d-product-requirements-v1.md`。
2. 再按本清单逐项实现，不允许跳项直接改主流程。
3. 每完成一项，必须更新状态为 `[x]` 并补验收记录。
4. 每次非 trivial 改动必须：
- 代码实现
- 最低执行 `npm run check`
- 文档同步
- `git commit` + `git push`
- 提交信息使用中文

## 2. 阶段推进看板

状态图例：`[ ]` 未开始，`[~]` 进行中，`[x]` 已完成，`[!]` 阻塞

### G0. 文档与流程治理
- [x] G0-1 产出需求文档并纳入入口索引
- [x] G0-2 产出开发清单并定义勾选推进规则
- [x] G0-3 产出协作总结文档并固化提交流程
- [x] G0-4 创建本地 skill：`aideskpet-delivery-governance`

验收：
1. 三份文档可达且读取顺序清晰。

3. 提交后完成 push。

### A. 基线与入口切换
- [ ] A1 固化新主线入口（Qt6+Godot），移除旧运行时默认入口
- [ ] A2 输出纯新主线构建入口（非 npm 主入口）
- [ ] A3 文档入口同步（README + document-authority）

验收：
1. 默认启动链路不经过旧 Electron 入口。
2. 新入口命令可在本地成功执行。

### B. IPC 契约与进程职责
- [ ] B1 完整补齐 IPC v1 事件集合（含多宠物/角色切换/专注计时/成长同步）
- [ ] B2 Godot/Qt 双进程职责边界实现并校验
- [ ] B3 契约校验脚本覆盖新增事件

验收：
1. 所有消息包含公共字段。
2. 新增事件可收发并有失败兜底。

### C. 3D 角色与动作系统
- [ ] C1 默认角色 3D 动画状态机（locomotion + interaction + emotion）
- [ ] C2 其余角色 3D 化并支持切换
- [ ] C3 走路自然化（加减速/转向倾斜/停步恢复）
- [ ] C4 漫游系统（屏内巡游 + 离屏 + 随机回归）

验收：
1. 全角色可切换。
2. 行走与转向无明显漂浮感。
3. 离屏/回归可稳定复现。

### D. 玩法全量等价
- [ ] D1 单击/双击/拖拽语义迁移
- [ ] D2 四动作菜单闭环（聊天/喂食/抚摸/清洁）
- [ ] D3 成长与照顾系统迁移
- [ ] D4 专注模式与主动行为迁移
- [ ] D5 天气与系统联动迁移
- [ ] D6 多宠物管理迁移

验收：
1. 与旧版玩法语义一致。
2. 每项玩法有对应回归用例。

### E. 聊天与语音（离线优先）
- [ ] E1 离线命令词识别接入
- [ ] E2 离线 TTS 接入
- [ ] E3 在线 provider 增强与失败回退
- [ ] E4 异步链路压测（不阻塞渲染）

验收：
1. 无网时仍可进行基础交互。
2. 在线失败自动回退离线回复。

### F. 打包与发布（Mac only，dmg-only）
- [ ] F1 CMake + Godot export 打包链路落地
- [ ] F2 dmg-only 产物归档规范固定
- [ ] F3 打包后自动冒烟（挂载/启动/交互/卸载）
- [ ] F4 删除 pkg 产线与无用旧产物

验收：
1. 仅输出 dmg。
2. 安装后可见可交互。

### G. 稳定性与性能门槛
- [ ] G1 30 分钟稳定性验证
- [ ] G2 CPU/内存/启动时间指标达标
- [ ] G3 回归报告与发布候选记录

验收：
1. 包体 `<= 140MB`
2. 空闲 CPU `<= 6%`
3. 互动 CPU `<= 12%`
4. 常驻内存 `<= 160MB`
5. 30 分钟内存增量 `<= 60MB`
6. 启动到可交互 `<= 4s`
