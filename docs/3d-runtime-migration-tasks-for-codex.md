# AI Desk Pet 3D 换轨开发任务书（Codex 执行版）

## 1. 任务说明

本任务书用于指导 Codex 将 AI Desk Pet 从 Electron 2.5D 架构迁移到 Qt6 + Godot 混合架构。  
要求：玩法语义不变，视觉与性能优先，macOS 先行，达到门槛后尽快切换主线。

- 执行分支：`feat/3d-runtime-switch-a-bootstrap`
- 执行方式：按阶段推进，阶段验收通过后进入下一阶段

## 2. 总体交付与约束

1. 所有 non-trivial 变更必须包含：
   - 代码实现
   - 最低校验命令通过（保留现有 `npm run check`）
   - 文档同步
2. 本分支采用主线硬切换策略：Runtime3D 为唯一默认运行路径，不维护 legacy 运行时回切。
3. 迁移期新增检查脚本：
   - `npm run check`（现有）
   - `npm run check:runtime3d`（新增）
4. 所有新模块必须有最小可测入口和失败兜底路径。

## 3. 阶段任务拆解

### 阶段 A：仓库骨架与基线（P0）

任务 A1：
- 建立新目录结构：
  - `runtime/godot`
  - `runtime/qt-sidecar`
  - `runtime/shared-ipc`
  - `runtime/migration`
- 统一默认入口到 Runtime3D bootstrap，不再以 legacy 入口作为默认路径。

验收：
- 新目录存在且可独立执行最小启动命令。
- `npm run check` 继续通过。

任务 A2：
- 增加 ADR 文档（架构决策记录）：
  - Godot 主渲染 + Qt 侧车
  - IPC 协议版本策略
  - 数据迁移策略
- 输出基线性能报告（当前 legacy 包体、CPU、内存）。

验收：
- ADR 文档完整。
- 基线报告可复现。

### 阶段 B：运行时主链路（P0）

任务 B1：
- 实现 Godot 透明悬浮窗口与可点击区域控制。
- 实现 Qt 托盘、热键、显隐控制。

验收：
- macOS 上可显示/隐藏宠物窗口。
- 托盘与热键可用。

任务 B2：
- 打通 Qt ↔ Godot IPC 桥接。
- 完成 v1 事件集合：
  - app.show/app.hide/app.quit
  - pet.action/pet.voice_wakeup
  - settings.get/settings.set

验收：
- IPC 往返稳定，无死锁。
- 所有消息含公共字段（request_id/schema_version/timestamp/source/target）。

### 阶段 C：3D 默认机器人与动效（P0）

任务 C1：
- 构建默认机器人 3D 资产与动画控制器。
- 动画状态机实现：
  - idle/start_walk/walk/turn/stop
  - listen/celebrate/drop/pet/feed/clean

验收：
- 动画切换无明显跳变。
- 走路具有加减速、转向预备、身体倾斜、停步恢复。

任务 C2：
- 实现漫游系统：
  - 屏幕内巡游
  - 完全离屏
  - 随机边缘回归

验收：
- 可观察离屏行为和回归行为。
- 不出现长时间“丢宠物”状态。

### 阶段 D：核心互动闭环（P0）

任务 D1：
- 迁移四动作菜单与宠物点击交互。
- 保留单击、双击、拖拽语义。

验收：
- 操作路径与旧版一致（语义一致，不要求 UI 完全一致）。

任务 D2：
- 迁移基础聊天链路（最小可用）。
- 保留 provider 配置与限流语义。

验收：
- 可完成一轮用户输入 -> 回复输出。
- 错误场景有人格化兜底提示。

任务 D3：
- 本地语音优先实现：
  - 本地命令词识别
  - 本地 TTS 播放

验收：
- 命令词可触发对应动作。
- 语音链路不阻塞动画主循环。

### 阶段 E：数据迁移与切换（P0）

任务 E1：
- 编写迁移器：
  - legacy 数据读取
  - runtime3d 数据写入
  - 迁移前快照
  - 幂等处理
- 覆盖数据：
  - 设置、状态、成长、聊天历史、记忆

验收：
- 三类样本迁移成功：
  - 空数据用户
  - 普通用户
  - 重度用户
- 迁移失败可通过快照恢复数据并重跑迁移，不提供运行时回切。

任务 E2：
- 主线硬切换：
  - 默认入口固定为 runtime3d
  - legacy 代码仅作历史参考，不再提供运行时切换开关

验收：
- 默认入口稳定指向 runtime3d。
- 无隐藏回切路径。

### 阶段 F：性能收口（P0）

任务 F1：
- 3D 资源优化：
  - 模型面数预算
  - 纹理压缩与图集
  - 动画采样率收敛
- 低功耗模式：
  - 压力高时降频与关特效

验收：
- 达到首版指标：
  - 包体 <= 140MB
  - 空闲 CPU <= 6%
  - 互动 CPU <= 12%
  - 内存 <= 160MB
  - 30 分钟内存增量 <= 60MB

任务 F2：
- 构建与发布脚本（macOS）：
  - 新运行时构建命令
  - 产物归档
  - 版本标识与日志

验收：
- 本地可稳定产出 mac 安装包。
- 产物可安装、可启动、可互动。

### 阶段 G：切换后回填（P1/P2）

任务 G1（P1）：
- 回填成长/照顾、主动行为、专注模式、天气/系统联动。

任务 G2（P1）：
- 回填长期记忆、跨会话持久化、多宠物。

任务 G3（P2）：
- 视觉细化、角色扩展、高级动效。

验收：
- 每次回填不得回退性能与包体门槛。
- 每个批次附回归记录与风险说明。

## 4. 任务优先级与依赖

1. 先做：A -> B -> C -> D -> E -> F
2. 切换后：G1 -> G2 -> G3
3. 依赖规则：
   - IPC 未稳定，不允许进入 D/E。
   - 数据迁移未稳定，不允许主线切换。
   - 性能不达标，不允许发布候选。

## 5. 测试与验收清单

1. 自动校验：
   - `npm run check`
   - `npm run check:runtime3d`
2. 手工冒烟（每个阶段）：
   - 启动、显隐、托盘、热键
   - 交互动作
   - 聊天与语音
   - 漫游与离屏回归
3. 稳定性：
   - 30 分钟连续运行
   - 关键链路无崩溃、无卡死、无失焦异常
4. 兼容性（首阶段）：
   - macOS Intel/Apple Silicon 各一台

## 6. PR 与提交规范（Codex 执行要求）

1. 提交粒度：按任务号提交（A1、A2、B1…）。
2. 每个 PR 必须包含：
   - 变更说明
   - 验证命令结果
   - 风险与数据修复说明
   - 文档同步
3. 禁止在同一 PR 混入无关改动。
4. 每个阶段结束输出阶段报告（通过、阻塞、下一步）。

## 7. 完成定义（Final DoD）

1. runtime3d 成为默认主线运行时（macOS）。
2. 主线切换门槛全部达标。
3. 关键用户路径可用：
   - 宠物可见可交互
   - 默认机器人 3D 自然行走
   - 可离屏回归
   - 聊天与本地语音可用
4. 不保留 legacy 运行时回切路径，仅保留迁移快照与修复工具。
5. 文档与任务记录完整。
