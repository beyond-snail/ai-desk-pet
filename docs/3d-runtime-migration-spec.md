# AI Desk Pet 3D 技术换轨方案（Qt6 + Godot）

## 1. 文档定位

本文档定义 AI Desk Pet 从当前 Electron 2.5D 实现迁移到 Qt6 + Godot 混合架构的完整技术方案。  
目标是保持玩法语义不变，优先解决包体偏大与运行功耗偏高问题，并在 macOS 先完成可切换版本。

- 文档状态：当前有效（迁移主方案）
- 适用版本：3D Runtime Migration v1
- 首发平台：macOS（Intel + Apple Silicon）

## 2. 目标与约束

### 2.1 迁移目标

1. 技术栈从 Electron 主运行时迁移为 Godot 主渲染 + Qt6 侧车。
2. 保持核心玩法语义不变：
   - 聊天、喂食、抚摸、清洁
   - 成长与状态衰减
   - 语音交互
   - 主动行为
   - 专注模式
   - 天气与系统联动
3. 默认机器人升级为 3D 角色，走路动作自然，有加减速与转向动感。
4. 支持屏幕内随机巡游 + 完全离屏 + 随机边缘回归。
5. 显著降低包体和运行功耗。

### 2.2 迁移策略

1. 研发策略：分阶段研发，达到切换门槛后尽快切换主线。
2. 产品策略：视觉与性能优先，玩法按优先级回填。
3. 平台策略：macOS 先行，Windows/Linux 后续跟进。

### 2.3 非目标（本阶段不做）

1. 不追求首版三端同时上线。
2. 不做角色商城、支付、云同步等商业化能力。
3. 不做多风格 3D 渲染管线（先固定一套轻量管线）。

## 3. 架构方案

### 3.1 总体架构

1. Godot Runtime（主进程职责）：
   - 透明悬浮窗口、3D 角色渲染、动画状态机
   - 漫游/离屏回归逻辑
   - 互动 UI（快捷动作、聊天面板、状态可视化）
   - 玩法核心状态机（care/growth/focus/proactive）
2. Qt Sidecar（系统能力职责）：
   - 托盘菜单
   - 全局热键
   - 自启动
   - 系统通知与平台差异适配
3. IPC Bridge（通信层）：
   - Qt 和 Godot 之间使用本地 IPC
   - 统一消息协议与版本管理
4. Legacy Compatibility（兼容层）：
   - 旧数据读取、迁移、回滚快照

### 3.2 进程拓扑（固定决策）

采用：Godot 主渲染 + Qt 侧车（独立进程）。  
不采用：Qt 主窗体嵌 Godot。  
理由：跨平台透明穿透窗口行为更稳定，便于单独压测渲染功耗。

### 3.3 模块边界

1. Godot 只处理“宠物表现 + 玩法逻辑 + 前端交互”。
2. Qt 只处理“系统集成能力”。
3. AI 调用、语音能力、天气与系统监控走服务接口，不耦合到渲染节点。
4. 任何跨进程行为必须通过 IPC 契约，不允许隐式共享状态。

## 4. 核心技术设计

### 4.1 窗口与交互

1. 透明、无边框、置顶窗口。
2. 鼠标穿透策略：
   - 默认穿透
   - 宠物/UI 区域动态可点
3. 显隐与唤醒：
   - 托盘显示/隐藏
   - 全局热键唤醒语音
4. 保持当前用户心智：交互区可点，非交互区不打扰桌面操作。

### 4.2 动画与移动（默认机器人重点）

1. 动画状态机：
   - locomotion: idle, start_walk, walk, turn, stop
   - interaction: listen, celebrate, drop, pet, feed, clean
   - emotion overlay: happy, sleepy, hungry, sad, excited
2. 移动模型：
   - 速度曲线：加速、匀速、减速
   - 转向：预备减速 + 身体倾斜 + 转身恢复
   - 触地冲击：轻微上下压缩
3. 路径策略：
   - 屏幕内随机目标
   - 可完全离屏
   - 离屏后随机停留并从边缘回归
4. 低功耗降级：
   - 压力高时降低动画更新频率与粒子效果
   - 优先保持动作连贯，次要效果可关闭

### 4.3 语音策略（全本地优先）

1. 本地命令词识别优先，用于交互触发（你好、过来、走开、喂食等）。
2. 本地 TTS 优先。
3. 平台能力不稳定时允许单平台兜底，但默认仍以本地方案为准。
4. 语音链路必须异步，不阻塞动画与输入主循环。

### 4.4 数据与兼容

1. 迁移旧存储数据：
   - 用户设置
   - 宠物状态
   - 成长进度
   - 聊天历史与记忆
2. 迁移原则：
   - 一次迁移、幂等可重复
   - 迁移前自动备份
   - 可回滚到 legacy 版本
3. 数据命名空间：
   - `legacy.*` 只读
   - `runtime3d.*` 新写入

## 5. IPC 契约（固定）

### 5.1 消息公共字段

每条消息必须包含：
- request_id
- schema_version
- timestamp
- source
- target

### 5.2 事件集合（v1）

1. App 生命周期：
   - app.show
   - app.hide
   - app.quit
2. 宠物控制：
   - pet.action
   - pet.voice_wakeup
   - pet.focus_mode
3. 设置读写：
   - settings.get
   - settings.set
4. 聊天链路：
   - chat.request
   - chat.stream_chunk
   - chat.done
   - chat.error
5. 语音链路：
   - speech.listen.start
   - speech.listen.stop
   - speech.tts.speak
6. 系统事件：
   - system.metrics.push

## 6. 切换门槛与阶段

### 6.1 主线切换门槛（必须满足）

1. 默认机器人 3D 动作可用，走路自然，离屏回归可用。
2. 托盘、热键、显隐、基础语音可用。
3. 四动作菜单 + 基础聊天可用。
4. 数据迁移成功率 100%（测试样本集）。
5. macOS 构建稳定可安装，30 分钟连续运行无崩溃。

### 6.2 切换后回填顺序

1. P0：成长/照顾规则、状态可视化、主动行为、专注模式、天气与系统联动。
2. P1：长期记忆、跨会话持久化、多宠物。
3. P2：高级动效、角色扩展与视觉细化。

## 7. 性能与包体指标（macOS 首版）

1. 安装包体积：<= 140MB（首版），后续优化目标 <= 120MB。
2. 空闲 CPU：<= 6%。
3. 常规互动 CPU：<= 12%。
4. 常驻内存：<= 160MB。
5. 连续运行 30 分钟内存增长：<= 60MB。
6. 启动到可交互：<= 4s。

## 8. 风险与回滚

1. 风险：透明穿透跨平台差异。  
   策略：先锁 macOS，Windows/Linux 单独适配层收敛。
2. 风险：本地语音模型导致 CPU 峰值。  
   策略：命令词轻量模型 + 异步队列 + 降频策略。
3. 风险：一次切换导致玩法缺口。  
   策略：切换门槛强约束 + 切换后 P0 快速补齐。
4. 风险：迁移数据异常。  
   策略：迁移前快照 + 自动回滚开关。

## 9. 交付定义（DoD）

1. 代码：新运行时主链路可运行，模块职责清晰。
2. 验证：自动检查通过，关键手工回归通过。
3. 文档：技术方案、任务文档、迁移记录同步更新。
4. 可回滚：保留 legacy 可构建能力与数据回滚入口。
