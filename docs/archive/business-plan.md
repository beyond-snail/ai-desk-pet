## 一、产品定位

### 1.1 一句话定义

**AI Desk Pet** — 一只住在你桌面上的 AI 宠物，能聊天、会成长、懂你的工作节奏。

### 1.2 核心价值主张

| 维度 | 传统桌宠 | 手机电子宠物 | AI Desk Pet |
|------|---------|-------------|-------------|
| 智能对话 | 无 | 预设脚本 | LLM 驱动，真正理解语义 |
| 情感连接 | 纯装饰 | 养成但无深度 | 养成 + 情绪感知 + 主动关怀 |
| 实用性 | 无 | 无 | 系统监控、番茄钟、天气、提醒 |
| 存在感 | 小窗口 | 需打开 App | 全屏透明覆盖，始终陪伴 |
| 成长性 | 无 | 有限 | 10 级进化 + 能力解锁 + 多角色 |

### 1.3 目标用户画像

**主力用户：Z 世代 & 千禧一代桌面办公/创作者**

| 画像 | 描述 | 核心需求 |
|------|------|---------|
| 独居程序员 | 长时间面对电脑，社交少 | 陪伴感、工作提醒、减压 |
| 设计师/创作者 | 审美敏感，喜欢个性化 | 好看的角色、自定义皮肤 |
| 学生党 | 需要专注学习，容易分心 | 番茄钟、学习陪伴、情绪调节 |
| 二次元/宠物爱好者 | 对虚拟角色有情感投射 | 养成乐趣、收集角色、社交分享 |
| 效率工具用户 | 追求桌面工具整合 | 系统监控、快捷操作、天气查询 |

### 1.4 竞品分析

| 产品 | 平台 | 定价 | 优势 | 劣势 |
|------|------|------|------|------|
| Shimeji | Windows | 免费 | 社区角色丰富 | 无 AI、无养成、Java 依赖 |
| Desktop Goose | Win/Mac | $5 | 搞笑、病毒传播 | 纯恶搞无深度、无对话 |
| 拓麻歌子 App | 移动端 | 免费+内购 | 经典 IP、养成深度 | 非桌面端、需主动打开 |
| Replika | 移动端 | 订阅$20/月 | AI 对话深度 | 无桌面端、无宠物养成 |
| **AI Desk Pet** | **桌面端** | **见下方** | **AI+养成+桌面融合** | **新品牌，需建立认知** |

**差异化壁垒**：市场上没有同时具备「LLM 对话 + 宠物养成 + 桌面原生」三要素的产品。

## 二、商业模式设计

### 2.1 推荐模式：免费核心 + 角色商城 + 可选订阅

经过分析，推荐采用**三层变现**结构：

```
┌─────────────────────────────────────────────┐
│  第三层：Pro 订阅 ($4.99/月 或 $39.99/年)    │
│  高级 AI 人格、云同步、多设备、优先更新       │
├─────────────────────────────────────────────┤
│  第二层：角色商城 (单个 $0.99-$2.99)          │
│  付费角色/皮肤、限定主题、社区创作者分成       │
├─────────────────────────────────────────────┤
│  第一层：免费核心 (永久免费)                   │
│  3 内置角色、AI 对话、养成系统、基础功能全开    │
└─────────────────────────────────────────────┘
```

### 2.2 为什么不选其他模式

| 模式 | 不推荐原因 |
|------|-----------|
| 纯买断 | 桌宠品类用户付费意愿低，$5-10 的买断价难以覆盖 LLM API 成本 |
| 纯订阅 | 门槛太高，桌宠不是刚需，用户会流失 |
| 广告变现 | 桌宠场景放广告体验极差，破坏陪伴感 |

### 2.3 免费层详情（获客引擎）

免费用户获得完整体验，不做功能阉割：

- 3 个内置角色（毛毛虫、赛博机器人、像素宠物）
- AI 对话（使用 DeepSeek，成本极低约 ¥0.001/次对话）
- 完整养成系统（10 级成长、情绪、喂养、清洁）
- 番茄钟、天气、系统监控
- 屏幕边缘互动、天气粒子效果

**设计哲学**：免费层必须足够好，让用户产生情感连接后自然付费。

### 2.4 角色商城详情（核心变现）

这是主要收入来源，利用用户对角色的情感投射：

| 类别 | 价格 | 示例 |
|------|------|------|
| 标准角色 | $0.99 | 小狐狸、企鹅、仓鼠 |
| 精品角色 | $1.99 | 龙宝宝、独角兽（含独特进化路线） |
| 限定角色 | $2.99 | 节日限定（圣诞驯鹿、春节龙）、联名角色 |
| 皮肤包 | $0.49 | 现有角色的配色变体 |
| 主题包 | $1.99 | 角色+配套天气效果+聊天气泡样式 |

**创作者经济**：
- 开放角色创作工具（CSS+HTML 模板）
- 创作者上传角色到商城，获得 70% 分成
- 社区投票决定精选角色
- 这是长期护城河——用户创造内容，平台获得无限角色供给

### 2.5 Pro 订阅详情（高价值用户）

| 功能 | 说明 |
|------|------|
| 高级 AI 人格 | 解锁更多对话风格（毒舌、温柔、学术、搞笑） |
| 云同步 | 宠物状态跨设备同步 |
| 多宠物 | 同时养 3+ 只宠物（免费版限 1 只） |
| 优先更新 | 提前体验新角色和功能 |
| 自定义 LLM | 接入自己的 GPT-4/Claude API Key |
| 去除水印 | 分享截图无水印 |

### 2.6 LLM 成本控制策略

这是关键——AI 对话是核心功能但也是主要成本：

| 策略 | 说明 |
|------|------|
| 默认 DeepSeek | 成本约 ¥1/百万 token，极低 |
| 本地缓存 | 常见问候/反应使用预设回复，不调 API |
| 对话频率限制 | 免费用户 50 次/天（足够日常），Pro 无限 |
| 短上下文 | 保留最近 10 条对话（非 20 条），降低 token 消耗 |
| 主动行为离线化 | 宠物的主动行为（提醒、问候）使用本地模板，不调 LLM |

预估单用户月均 LLM 成本：约 ¥0.3-0.5（DeepSeek），完全可控。

## 三、增长策略

### 3.1 冷启动（0→10,000 用户）

**阶段目标**：验证产品市场匹配度（PMF）

| 渠道 | 策略 | 预期效果 |
|------|------|---------|
| Product Hunt | 首发上线，准备 GIF 演示 | 500-2000 首日访问 |
| Reddit | r/desktoppets, r/programming, r/coolgithubprojects | 社区种子用户 |
| GitHub | 开源核心引擎，付费角色商城 | 开发者信任 + SEO |
| 小红书/B站 | 桌宠演示视频，"程序员的电子宠物" | 国内 Z 世代触达 |
| V2EX/掘金 | 技术分享 + 产品发布 | 开发者社区 |
| Twitter/X | #desktoppet #AIcompanion 话题 | 全球开发者圈 |

**关键动作**：
1. 制作 30 秒演示 GIF（宠物在桌面爬行 → 点击对话 → 喂食 → 进化）
2. 准备 Product Hunt 发布页（英文）
3. B 站录制 3 分钟演示视频（中文）

### 3.2 增长期（10,000→100,000 用户）

| 策略 | 说明 |
|------|------|
| 角色创作者计划 | 邀请像素画/CSS 艺术家创作角色，分成激励 |
| 社交裂变 | "分享你的宠物截图" 活动，宠物截图带产品水印 |
| 节日运营 | 每个节日推出限定角色（制造稀缺感和 FOMO） |
| KOL 合作 | 找 B 站/YouTube 科技区 UP 主体验评测 |
| SEO | "desktop pet"、"AI companion"、"桌面宠物" 关键词优化 |
| 口碑传播 | 宠物养到高等级后解锁"邀请好友"功能，双方获得限定皮肤 |

### 3.3 成熟期（100,000+ 用户）

- 开放 API/插件系统，构建开发者生态
- 推出移动端伴侣 App（v2.0 Flutter）
- 探索 B2B 场景（企业定制吉祥物桌宠）
- 考虑 IP 联名（游戏角色、动漫角色授权）

## 四、全球化策略

### 4.1 多语言优先级

| 优先级 | 语言 | 原因 |
|--------|------|------|
| P0 | 中文 + 英文 | 同步上线，覆盖最大市场 |
| P1 | 日文 | 桌宠/养成文化最浓厚的市场 |
| P2 | 韩文 | 类似日本，虚拟角色接受度高 |
| P3 | 西班牙语/葡萄牙语 | 拉美市场增长快 |

### 4.2 分发渠道

| 平台 | 地区 | 说明 |
|------|------|------|
| 官网直下载 | 全球 | 主渠道，保留最高利润 |
| GitHub Releases | 全球 | 开发者渠道 |
| Homebrew | 欧美 | macOS 用户习惯 |
| Microsoft Store | 全球 | Windows 用户发现 |
| Steam | 全球 | 游戏化桌宠有先例（Shimeji） |
| 少数派/数码荔枝 | 中国 | 国内独立软件分发 |

### 4.3 支付方案

| 地区 | 支付方式 |
|------|---------|
| 全球 | Stripe（信用卡/Apple Pay/Google Pay） |
| 中国 | 微信支付 + 支付宝 |
| 备选 | Paddle（处理全球税务合规） |

## 五、财务模型

### 5.1 成本结构（月度）

| 项目 | 初期(1K用户) | 增长期(50K用户) | 成熟期(200K用户) |
|------|-------------|----------------|-----------------|
| LLM API (DeepSeek) | ¥300 | ¥15,000 | ¥60,000 |
| 服务器（角色商城+云同步） | ¥200 | ¥2,000 | ¥8,000 |
| 域名+CDN | ¥100 | ¥500 | ¥2,000 |
| Apple 开发者账号 | ¥688/年 | - | - |
| **月度总成本** | **~¥800** | **~¥17,500** | **~¥70,000** |

### 5.2 收入预测

假设转化率基于行业基准：
- 免费→付费角色购买：5-8% 用户
- 免费→Pro 订阅：2-3% 用户
- 角色平均客单价：$1.5（约 ¥10）
- Pro 订阅：$4.99/月（约 ¥35/月）

| 阶段 | MAU | 角色收入/月 | 订阅收入/月 | 总收入/月 |
|------|-----|-----------|-----------|----------|
| 初期 | 1,000 | ¥500 | ¥700 | ¥1,200 |
| 增长期 | 50,000 | ¥25,000 | ¥35,000 | ¥60,000 |
| 成熟期 | 200,000 | ¥100,000 | ¥140,000 | ¥240,000 |

### 5.3 盈亏平衡点

- 初期几乎零成本运营（DeepSeek 极便宜，Electron 无服务器依赖）
- 预计 **5,000 MAU** 时达到盈亏平衡
- 关键指标：LTV（用户生命周期价值）> CAC（获客成本）× 3

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| LLM API 涨价 | 中 | 高 | 多供应商切换能力已内置；探索本地小模型 |
| 用户新鲜感消退 | 高 | 中 | 持续角色更新 + 节日运营 + 成长系统延长生命周期 |
| 竞品模仿 | 中 | 中 | 先发优势 + 创作者生态护城河 + 快速迭代 |
| 平台政策变化 | 低 | 高 | 多渠道分发，不依赖单一平台 |
| macOS 安全限制收紧 | 中 | 中 | 保持 Apple 开发者签名，遵循沙盒规范 |

## 八、核心指标（North Star Metrics）

| 指标 | 定义 | 目标 |
|------|------|------|
| DAU/MAU | 日活/月活比 | > 40%（说明用户每天打开） |
| 平均养成天数 | 用户持续养宠物的天数 | > 14 天 |
| 对话次数/日 | 每用户每天与宠物对话次数 | > 3 次 |
| 角色购买率 | 付费用户占比 | > 5% |
| NPS | 净推荐值 | > 50 |
