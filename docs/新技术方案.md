# 跨Windows/Mac/Linux三端的桌面悬浮3D机器人宠物 完整技术方案
## 一、方案核心调整（适配三端）
本方案在原有核心功能基础上，重点解决**跨Windows/Mac/Linux三端兼容**问题，核心原则是：「底层模块做系统适配、核心逻辑抽象解耦、统一技术栈降低适配成本」，确保3D机器人宠物在三大桌面系统下均能实现「悬浮显示、自由移动、语音交互」核心功能。

### 核心目标（新增跨端要求）
1. 三端统一功能：悬浮透明窗口、3D机器人移动/动画、中文语音交互；
2. 跨端兼容标准：
   - Windows：Win10/11（64位）；
   - Mac：macOS 12+（Intel/Apple Silicon）；
   - Linux：Ubuntu 20.04+/CentOS 8+（主流发行版）；
3. 跨端体验一致性：功能无删减、操作逻辑统一、性能指标对齐（内存＜100MB，响应＜2s）。

## 二、跨端技术选型（核心调整）
### 1. 三端统一技术栈（优先跨端原生支持）
| 功能模块                | 跨端技术选型                          | 各系统适配说明                          |
|-------------------------|---------------------------------------|-----------------------------------------|
| 桌面窗口层（核心）| Qt6（PyQt6/Qt C++）> Electron        | Qt6原生支持三端窗口定制，适配成本最低；Electron重但兼容性强 |
| 3D渲染/动画层           | Godot Engine > Ursina（仅Python）> Unity | Godot原生跨三端，打包体积小（＜50MB）；Ursina仅适配Windows/Linux，Mac需额外适配 |
| 语音识别层              | OpenAI Whisper（本地）> 讯飞开放平台（联网） | Whisper通过Python跨三端，本地部署无系统依赖；讯飞需联网但识别率稳定 |
| 语音合成层              | Edge TTS（联网）> pyttsx3（仅Windows/Linux） | Edge TTS跨三端（需联网）；pyttsx3在Mac需替换为pyobjc |
| 3D模型/动画             | GLB通用格式 + Mixamo动画库            | GLB为跨引擎通用格式，三端加载无差异      |

### 2. 各系统核心痛点适配方案
| 系统       | 核心痛点（悬浮/穿透/渲染）| 适配技术方案                          |
|------------|---------------------------------------|---------------------------------------|
| Windows    | 窗口穿透/分层属性                     | 调用Win32 API：`WS_EX_LAYERED`+`WS_EX_TRANSPARENT`，Qt6封装接口直接调用 |
| Mac        | 窗口透明/置顶/鼠标穿透                | 调用macOS原生API：`NSWindow`设置`backgroundColor=clear`+`ignoresMouseEvents`，Qt6封装`QMacCocoaViewContainer` |
| Linux      | 窗口透明/穿透（不同桌面环境差异）| 针对X11/Wayland分别适配：X11用`NET_WM_WINDOW_TYPE_DOCK`，Wayland用`xdg-decoration`协议 |

## 三、跨端架构设计（核心：抽象解耦）
### 1. 整体架构（新增跨端适配层）
```mermaid
graph TD
    A[用户交互层] --> B[核心逻辑层]
    B --> C[跨端适配层]
    C --> D[系统底层层]
    A包括1[鼠标交互]、2[语音输入]、3[3D渲染输出]、4[语音输出]
    B包括1[窗口管理抽象模块]、2[3D角色控制模块]、3[移动逻辑模块]、4[语音处理模块]
    C包括1[Windows适配子模块]、2[Mac适配子模块]、3[Linux适配子模块]
    D包括1[各系统窗口API]、2[各系统音频驱动]、3[各系统3D渲染接口]
```

### 2. 核心解耦设计（关键跨端手段）
#### （1）窗口管理抽象层
- 定义统一的窗口接口（`IWindow`）：包含`create()`/`setTransparent()`/`setClickThrough()`/`setTop()`等通用方法；
- 各系统实现具体子类（`WindowsWindow`/`MacWindow`/`LinuxWindow`），核心逻辑示例：
  ```python
  # 抽象接口
  class IWindow:
      def create(self):
          pass
      def set_transparent(self, enable: bool):
          pass
      def set_click_through(self, enable: bool):
          pass

  # Windows实现
  class WindowsWindow(IWindow):
      def set_click_through(self, enable: bool):
          # 调用Win32 API设置穿透
          import ctypes
          hwnd = self.window.winId()
          ctypes.windll.user32.SetWindowLongPtrW(hwnd, -20, 0x80000 | 0x20)

  # Mac实现
  class MacWindow(IWindow):
      def set_click_through(self, enable: bool):
          # 调用macOS Cocoa API
          from PyQt6.QtWidgets import QWidget
          self.window.setAttribute(Qt.WidgetAttribute.WA_MacAlwaysShowToolWindow, True)
          self.window.setWindowFlags(Qt.WindowType.FramelessWindowHint | Qt.WindowType.WindowStaysOnTopHint)
  ```

#### （2）语音模块跨端适配
- 语音识别：统一使用Whisper（Python版），三端均通过`whisper`库调用本地模型，无系统差异；
- 语音合成：抽象`ITTS`接口，各系统实现子类：
  - Windows/Linux：`pyttsx3`；
  - Mac：`pyobjc`调用系统`NSSpeechSynthesizer`；
  - 兜底方案：统一使用Edge TTS（联网），三端通过HTTP调用，无系统依赖。

#### （3）3D渲染跨端
- 选用Godot Engine作为3D渲染核心：
  - Godot原生支持三端打包，无需额外适配；
  - 统一使用GLB模型格式，三端加载逻辑完全一致；
  - 动画控制器（AnimationPlayer）三端行为统一，无需调整。

## 四、三端落地实施流程
### 1. 统一开发阶段
1. 基于Qt6+Godot搭建跨端项目框架，抽象核心接口（窗口/语音/渲染）；
2. 开发核心逻辑（移动/动画/语音匹配），确保逻辑与系统解耦；
3. 针对各系统编写适配层代码，仅适配层与系统交互，核心逻辑不改动。

### 2. 分系统调试阶段
| 系统       | 调试重点                          | 验证方法                          |
|------------|-----------------------------------|-----------------------------------|
| Windows    | 窗口穿透/分层属性、pyttsx3语音合成 | 测试窗口是否遮挡其他应用、语音是否正常播放 |
| Mac        | 窗口透明/鼠标穿透、Apple Silicon适配 | 测试M1/M2芯片下运行是否流畅、窗口是否穿透 |
| Linux      | 不同桌面环境（GNOME/KDE）适配     | 在Ubuntu/CentOS不同发行版测试功能完整性 |

### 3. 打包交付阶段
| 系统       | 打包工具                          | 交付格式                          |
|------------|-----------------------------------|-----------------------------------|
| Windows    | Godot Export/ PyInstaller         | EXE（单文件/绿色版）|
| Mac        | Godot Export/ py2app              | DMG（应用包）|
| Linux      | Godot Export/ PyInstaller         | AppImage（免安装）/DEB/RPM包       |

## 六、跨端风险与应对方案
| 潜在风险                          | 影响范围 | 应对方案                          |
|-----------------------------------|----------|-----------------------------------|
| Linux不同桌面环境（Wayland/X11）窗口适配差异 | Linux    | 优先适配X11（主流），Wayland提供降级方案（关闭穿透，仅保留透明） |
| Mac Apple Silicon芯片3D渲染性能问题 | Mac      | Godot启用Metal渲染后端，降低模型面数优化性能 |
| 语音合成在部分Linux发行版无系统语音包 | Linux    | 兜底使用Edge TTS联网合成          |
| Windows/Linux/Mac快捷键冲突       | 三端     | 抽象快捷键处理逻辑，各系统适配原生快捷键规则 |

### 总结
1. 跨三端核心是「抽象解耦+系统适配层」：核心逻辑与系统解耦，仅适配层对接各系统原生API，确保三端功能一致；
2. 技术选型优先Qt6+Godot：Qt6解决跨端窗口适配，Godot解决跨端3D渲染，二者结合是三端桌面应用的最优解；
3. 语音模块优先本地方案（Whisper）+ 联网兜底（Edge TTS），平衡体验与兼容性；
4. 落地时先完成核心逻辑开发，再分系统做适配调试，可大幅降低跨端开发成本。
