## 二、产品核心问题诊断

### 留存的本质

桌宠产品留住用户靠的是一个公式：

> **情感依附 = 存在感 × 记忆感 × 被需要感**

- **存在感**：宠物一直在桌面上，用户能看到它
- **记忆感**：宠物记得你，知道你叫什么、说过什么、做过什么
- **被需要感**：宠物的状态会变差，需要用户照顾

v1 三项都有基础，但都不够深。v2 指令在补记忆感和被需要感。接下来的规划继续沿这个方向深化。

### 当前最大的体验断层

1. **对话没有连续性**：每次打开 App，宠物不知道上次聊了什么（上下文只保留 10 条，且不跨会话持久化）
2. **宠物没有自己的故事**：用户不知道宠物的"过去"，情感投射无处落脚
3. **成长不可见**：等级在涨，但用户感知不到自己的宠物在"变化"
4. **互动太单一**：只有聊天、喂食、抚摸、清洁，缺乏新鲜感

## 三、版本规划

### v1.2 — 情感深化（2-3 周）

**主题：让用户感觉"这是我的宠物"**

#### 功能列表

**智能长期记忆系统（核心）**

这是桌宠智能化的基石。目标是让宠物像豆包一样真正"记得你"——不只是行为统计，而是理解你说过的事、你的习惯、你的状态。

核心机制：**Memory Extraction**

每次对话结束后，用一次轻量 LLM 调用提取本轮对话的关键信息，存入独立的"长期记忆库"。下次启动时，将记忆库摘要注入 system prompt。

三层记忆架构：

```
┌─────────────────────────────────────────────────┐
│  第一层：短期上下文（已有）                        │
│  最近 10 条对话，当前会话内有效                    │
├─────────────────────────────────────────────────┤
│  第二层：跨会话记忆（v1.2 新增）                   │
│  持久化最近 20 条对话到 store，重启后恢复           │
├─────────────────────────────────────────────────┤
│  第三层：长期记忆提取（v1.2 新增）                  │
│  AI 自动提取关键事实，存入结构化记忆库              │
│  例："用户叫小明"、"在做 AI 项目"、"喜欢喝咖啡"    │
└─────────────────────────────────────────────────┘
```

长期记忆提取规则：
- 每轮对话结束后，异步调用一次 LLM（不阻塞主流程）
- 提取 prompt：`"从以下对话中提取用户透露的关键个人信息，用一句话概括，不超过30字，如果没有新信息则返回空字符串"`
- 提取结果追加到记忆库（最多保留 30 条，按时间淘汰旧的）
- 记忆库注入 system prompt 时格式：`"关于用户你知道：[记忆1]；[记忆2]；..."`

成本控制：
- 提取调用使用最小模型（DeepSeek，约 ¥0.001/次）
- 只在用户主动发起对话时触发，主动行为不触发
- 内置用户每天对话次数限制已有，不额外增加成本

**跨会话对话持久化**
- 将最近 20 条对话内容持久化到 store（目前只在内存里，重启清空）
- 每次启动时加载，宠物能说出"上次你说……"

**宠物命名**
- 新手引导第一步改为：给宠物起名字
- 名字存入 store，注入 system prompt（"你叫 XX，是用户给你起的名字"）
- 宠物在对话中会用自己的名字自称

**成长日记**
- 每次升级/进化时，自动记录一条日记（时间 + 等级 + 当天互动摘要）
- 右键菜单新增"查看成长日记"入口
- 展示为简单的时间线列表（不需要复杂 UI）

**宠物心情可视化**
- 在宠物动画上叠加细微的表情变化（不改动画系统，用 CSS overlay 实现）
- 心情好：轻微发光效果；心情差：轻微灰度

#### 技术约束
- 不引入新依赖
- 成长日记用 store 存储，最多保留 50 条
- 命名在 OnboardingGuide 里加一步，不影响已有用户（已完成引导的跳过）

### v1.3 — 互动扩展（3-4 周）

**主题：让每天的互动有新鲜感**

#### 功能列表

**小游戏：猜谜/问答**
- 宠物每天主动发起一个小问题（天气、时间、随机冷知识）
- 用户回答后宠物给反馈，答对加经验值
- 实现方式：本地题库（50 题），不调 LLM，零成本

**心愿清单**
- 用户可以告诉宠物"今天想完成 XX"
- 宠物记录下来，在适当时机（番茄钟结束后、傍晚）主动询问进展
- 实现方式：存入 store，ProactiveBehavior 定时检查

**天气互动升级**
- 下雨天：宠物说"外面在下雨，要带伞吗？"并在桌面显示雨滴粒子效果
- 晴天：宠物偶尔说"今天天气不错，要不要出去走走？"
- 极端天气（高温/低温）：宠物有专属反应

**节日感知**
- 内置节日日历（春节、元旦、圣诞、情人节等 20 个节日）
- 节日当天宠物有专属问候和外观变化（CSS 装饰，不改动画系统）

#### 技术约束
- 小游戏题库用 JSON 文件存放在 `src/renderer/data/` 目录
- 节日日历同样用 JSON 文件
- 不引入新依赖

### v1.4 — 社交与分享（4-6 周）

**主题：让用户有理由告诉别人**

#### 功能列表

**宠物名片生成**
- 用户可以生成一张宠物名片（角色截图 + 名字 + 等级 + 养成天数 + 一句话）
- 用 Canvas API 生成图片，可保存/分享
- 名片样式简洁，带产品水印（"AI Desk Pet"）

**成就系统**
- 20 个成就（首次对话、养了 7 天、完成 10 次番茄钟、升到 5 级等）
- 解锁成就时 chatBubble 提示 + 记录到成长日记
- 右键菜单可查看成就列表

**邀请好友**
- 宠物养到 3 级后解锁"邀请好友"功能
- 生成邀请链接（指向官网/GitHub），好友安装后双方获得限定皮肤（本地解锁码机制）

#### 技术约束
- Canvas 名片生成纯前端实现
- 邀请码用本地哈希验证，不需要服务器
- 成就数据存入 store

### v2.0 — 角色商城（6-8 周）

**主题：建立变现基础**

这是产品从"工具"变成"平台"的关键版本。

#### 功能列表

**角色商城 UI**
- 在角色选择面板里新增"商城"入口
- 展示付费角色列表（图片 + 名称 + 价格 + 预览动画）
- 已购买角色直接解锁，未购买显示价格和购买按钮

**支付集成**
- 接入 Stripe（全球）+ 支付宝/微信（国内）
- 购买后本地存储解锁状态（加密）
- 支持礼品码兑换

**创作者工具 v1**
- 提供角色模板（HTML + CSS + JSON 规范）
- 文档说明如何创建自定义角色
- 支持本地加载自定义角色（设置里"加载本地角色"）

**Pro 订阅**
- 解锁：无限对话次数、云同步、多宠物（最多 3 只）、高级性格
- 订阅管理页面（查看状态、取消订阅）

#### 技术约束
- 商城数据从 CDN 拉取（JSON 格式），不需要复杂后端
- 支付回调需要简单后端（Node.js serverless function，Vercel 部署）
- 云同步用 Supabase（免费层够用）

## 四、体验优化 Backlog

以下是不属于某个版本、但需要持续改进的体验问题，按优先级排序：

### 高优先级

| 问题 | 现状 | 改进方向 |
|------|------|---------|
| 对话输入框体验 | 基础文本框 | 支持快捷键（Enter 发送，Shift+Enter 换行），输入时宠物有"思考"动画 |
| 宠物点击反馈 | 有动画但不够丰富 | 点击不同部位有不同反应（头部/身体/尾巴） |
| 设置面板 UX | 功能堆砌 | 分组折叠，常用设置置顶，高级设置收起 |
| 首次启动速度 | 有加载延迟 | 骨架屏或加载动画，避免白屏 |
| 错误提示 | 技术性错误信息 | 用宠物的口吻说错误（"我好像出了点问题……"） |

### 中优先级

| 问题 | 现状 | 改进方向 |
|------|------|---------|
| 多角色差异化 | 4 个角色说话方式一样 | 每个角色有独特的说话风格（在 character config 里定义） |
| 宠物移动路径 | 随机游走 | 偶尔走向用户鼠标位置，增加"跟随感" |
| 番茄钟结束提示 | chatBubble 文字 | 加声音提示（可选）+ 更明显的视觉反馈 |
| 天气粒子效果 | 已有但效果一般 | 雨天/雪天粒子更自然，与宠物动画联动 |

### 低优先级

| 问题 | 现状 | 改进方向 |
|------|------|---------|
| 深色/浅色模式 | 无 | 跟随系统主题自动切换 |
| 宠物音效 | 无 | 可选的轻音效（喂食、抚摸、升级） |
| 键盘快捷键 | 只有唤醒热键 | 更多快捷键（喂食、聊天、隐藏） |
| 多显示器支持 | 基础支持 | 宠物可以在多个显示器间移动 |

## 七、决策原则

给 Codex 和产品判断用的原则，遇到模糊需求时参考：

1. **情感优先于功能**：一个让用户感动的细节，比十个实用功能更有价值
2. **存在感优先于交互**：宠物在桌面上"活着"比功能丰富更重要
3. **不打扰原则**：宠物是陪伴者，不是打扰者。主动行为宁可少，不能烦
4. **渐进式复杂度**：新用户看到的是简单可爱，老用户发现的是深度
5. **零配置开箱**：任何功能的默认状态必须是"好用的"，配置是加分项不是前提
