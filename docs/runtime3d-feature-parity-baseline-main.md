# Runtime3D 与 main 功能对齐说明

## 1. 目的

回答“功能玩法是否变化”这个问题：  
技术栈可替换，但玩法语义不应改变。

## 2. 对齐原则

1. 保持功能语义一致。
2. 保持用户路径一致（可接受 UI 呈现差异）。
3. 技术实现可以完全重构。

## 3. 对齐清单（main -> runtime3d）

1. 角色系统 -> 保持角色可选与切换
2. 聊天链路 -> 保持输入/回复闭环
3. 四动作 -> 保持聊天/喂食/抚摸/清洁语义
4. 成长照顾 -> 保持状态衰减和成长推进
5. 专注模式 -> 保持开始/计时/结束
6. 主动行为 -> 保持低状态提醒和主动互动
7. 天气系统联动 -> 保持状态反馈与表现联动
8. 多宠物 -> 保持添加/管理/基础协同
9. 托盘热键 -> 保持显隐与唤醒入口

## 4. 当前结论

1. 功能目标：与 main 保持等价，不做缩减。
2. 当前阶段：cleanroom 先搭建新主线与占位 3D 表现，逐项回填并验收。
