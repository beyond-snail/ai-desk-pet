# AI Desk Pet Runtime3D 需求文档 v1

## 1. 文档定位

本文档定义 Runtime3D cleanroom 分支的需求基线：先明确“做什么”和“如何验收”。  
技术实现与任务拆解见开发清单文档。

固定口径：
1. 技术主线：`Qt6 + Godot`
2. 功能策略：玩法语义保持与 `main` 一致（不缩水）
3. 数据策略：不迁移旧数据，首次启动按新默认初始化
4. 安装形态：macOS 仅 `dmg`
5. 执行策略：`Mac 先行`

## 2. main 功能对齐基线

本需求要求“功能玩法不变”，对齐 `main` 当前能力基线（以 `01787aa` 及其之前能力集合为准），至少包含：

1. 角色系统与切换
2. 聊天与回复
3. 四动作（聊天/喂食/抚摸/清洁）
4. 情绪、照顾、成长
5. 专注模式
6. 主动行为
7. 天气与系统联动
8. 多宠物管理
9. 托盘与热键

## 3. 需求项与验收

### FR-01 可见可交互桌宠
1. 启动后可见桌宠。
2. 桌宠支持点击、拖拽、菜单动作。

验收：安装后 1 次启动可见且可交互。

### FR-02 玩法等价闭环
1. 保留 `main` 的核心玩法语义。
2. 用户路径保持一致（语义一致优先，不要求 UI 像素一致）。

验收：每个玩法项有对照回归结果。

### FR-03 3D 表现升级
1. 默认角色与可选角色支持 3D 表现。
2. 在正式 3D 模型到位前允许使用 SVG 占位 3D 方案。
3. 预留正式模型替换接口（模型档案、动画集标识、渲染数据口）。

验收：角色可切换，动作状态与交互反馈完整。

### FR-04 语音与聊天
1. 离线优先（命令词 + 本地 TTS + 本地回复兜底）。
2. 在线 provider 可配置，失败自动降级。

验收：离线可用、在线失败有兜底。

### FR-05 打包交付
1. 提供 macOS dmg 产物。
2. 安装后可直接启动并进入可交互状态。
3. 产物目录统一为 `dist/runtime3d-release/darwin-<arch>/AIDeskPet-runtime3d-darwin-<arch>-v<version>.dmg`。

验收：dmg 可挂载安装，应用可启动并跑通核心交互。

### FR-06 配置命名空间统一
1. 运行时配置统一到 `runtime3d.*` 命名空间。
2. 读取策略允许旧键只读回退，写入统一走 `runtime3d.*`。

验收：核心配置（角色/成长/照顾/聊天/专注/主动行为/天气/多宠物）均可在命名空间切换后正常工作。

## 4. 非功能门槛（Mac）

1. 包体 `<= 140MB`
2. 空闲 CPU `<= 6%`
3. 互动 CPU `<= 12%`
4. 常驻内存 `<= 160MB`
5. 30 分钟内存增长 `<= 60MB`
6. 启动到可交互 `<= 4s`
