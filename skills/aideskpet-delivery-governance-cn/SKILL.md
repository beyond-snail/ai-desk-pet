# AI Desk Pet 交付治理（中文版）

## 适用场景

在本仓库执行任意中大型任务时使用，特别是以下场景：

1. 新需求落地或大重构。
2. 长会话后需要防止偏航。
3. 需要强制“做完一项勾一项”的过程管理。
4. 需要确保每次改动有验证、文档同步、提交与推送。

## 固定流程

### 步骤 1：先写需求文档（PRD）

1. 先定义范围、目标、验收、非目标。
2. 明确“功能语义必须保持不变”的部分。
3. 如与现有实现冲突，先更新需求口径再改代码。

### 步骤 2：再写开发清单（可勾选）

1. 按阶段拆任务。
2. 使用 `[ ]`、`[~]`、`[x]`、`[!]` 标记状态。
3. 每项必须有验收命令和结果记录。

### 步骤 3：逐项实现并即时回写状态

1. 一次只推进一到两个清单项。
2. 实现后立即把该项改为 `[x]`。
3. 不允许“口头完成，文档不更新”。

### 步骤 4：验证、提交、推送

1. 执行仓库标准校验（本分支优先 `make check-runtime3d`）。
2. 非 trivial 改动必须有文档同步。
3. 每次完成后执行 `git commit` 与 `git push`。
4. commit message 使用中文，且绑定清单项。

### 步骤 5：防偏航策略

1. 若对话出现新方向，先更新需求文档。
2. 若清单与实现冲突，先修清单再继续。
3. 若遇阻塞，标记 `[!]` 并记录阻塞原因。

## 输出约定

每次阶段性反馈必须包含：

1. 完成的清单项 ID。
2. 执行的验证命令与结论。
3. 提交哈希、中文提交信息、push 结果。
4. 剩余待办和下一步。

## 资源

使用 `references/交付模板.md` 作为需求、清单、验证、提交记录的统一模板。
